Science Commons
Materials Transfer Agreement Chooser

Michael Travers, 11 July 2007



Introduction

Science Commons Materials Transfer Agreements (MTAs) are contracts that govern the transfer of tangible research materials from one research institution (the provider) to another (the recipient). They are most commonly employed in the transfer of "unique research resources" such as cell lines, monoclonal antibodies, reagents, animal models, growth factors, combinatorial chemistry and DNA libraries, clones and cloning tools (such as PCR), methods, laboratory equipment and machines. Non-biological and synthetic materials, such as certain nano-materials, chemical reagents, and chemical substrates may be shared under MTAs as well.

The MTA Chooser is a web-based tool for selecting an appropriate agreement for a material. There are two forms of the chooser: a standalone web page, and a popup wizard designed to be embedded in provider's web pages. This document is addressed chiefly to those who want to embed the popup wizard.

Terminology

Provider:

an institution or person making some materials available

Material:

a scientific material (ie, a tissue sample)

Agreement:

a generic legal document

Offer:

a specific offer of a material under an agreement



Agreement types

There are four basic types of agreement:

  1.  UBMTA (ubmta)

  2.  SLA (sla)

  3.  Science Commons (sc)

  4.  Custom

The Science Commons agreement has twelve variants with different restrictions. Each agreement type other than custom has an associated set of documents available through the MTA web site. Custom agreements have a user-specified URL.

URLs

Every agreement type defines a set of URLs.  These look like this:

http://mta.sciencecommons.org/agreements/<Agreement>/<Version>[/<DocumentType>][<Parameters>

For instance, the UBMTA deed is found at:

http://mta.sciencecommons.org/agreements/ubmta/1.0

and related documents are found at:

http://mta.sciencecommons.org/agreements/ubmta/1.0/legalcode

http://mta.sciencecommons.org/agreements/ubmta/1.0/letter



Legal values for <Agreement> are:

ubmta, sla, sc, sc[-rp|-df][-ns][-rd]

Where the options for SC mean:

-rp
Restricted to protocol
-df
Restricted to disease
-ns
No scaleup
-rd
Return or destroy


Legal values for <DocumentType> are:

blank

for deeds

legalcode

for the complete legal code

letter

for the implementing letter

Any of the URLs may contain optional URL parameters. The exact set of parameters is still being defined.

It is likely that other document types will be defined, i.e. for icons.

Chooser API

The popup version of the chooser may be called from Javascript. The caller can post event handlers that are called when the chooser has generated an offer.

Calling the chooser

This code creates a popup window with the chooser, and arranges for the onOfferEvent function to be called when an offer is complete:

    var wp = window.open("http://mta.sciencecommons.org/popup", "MTA popup",          "width=500,height=300,resizable=no,status=no");
    wp.onload = function() {
        wp.document.offerEvent.subscribe(onOfferEvent, wp.document);
    }

The event handler is passed an offer argument, which has an API defined below. 

onOfferEvent = function(type, args, me) { 
     var offer = args[0];
     ...
}

Offer object API

get_agreement_id()

Returns the agreement code (ie, 'sc-rp-ns')

get_name()

Get a human-readable name for the offer

get_version()

Return the version (currently always '1.0')

get_deed_uri()
get_implementing_uri()
get_legal_uri()

Return URIs for the various documents associated with the offer


get_metadata()

Returns a representation of the offer in HTML with RDFa embedded metadata

get_info()

Returns an object containing all parameters of the offer.
